## Weekly Challenges & Lessons

**Biggest challenge this week:**

The biggest challenge this week was learning how to use many new command-line tools and forensic techniques in the picoCTF environment. I often encountered unfamiliar commands and tools such as exiftool, strings, steghide, gunzip, packet analysis in Wireshark, and decoding tools like CyberChef. Because it was my first time working with some of these tools, I initially struggled to know which command or approach to try first when analyzing files or solving challenges.

**How you overcame it (or plan to):**

I overcame this challenge by searching for explanations, watching tutorials, and experimenting with the commands in the picoCTF webshell. Whenever I encountered a new command, I tried to research its function and understand how it works rather than just copying the solution. By repeatedly practicing commands like wget, ls, cat, strings, and decoding techniques such as Base64, I gradually became more comfortable with the command-line environment and forensic tools.

**Most valuable lesson learned:**

The most valuable lesson I learned is that digital evidence can be hidden in many forms, such as metadata, encoded text, steganography, corrupted files, or network packets. Tools like exiftool, strings, and steganography techniques such as LSB extraction can reveal hidden information. I also learned that analyzing files step-by-step and paying attention to small clues, such as unusual metadata or encoded data, is essential in solving digital forensics and security challenges.

**Adjustment for next week:**

For next week, I plan to practice more command-line tools and familiarize myself with common forensic commands before starting new challenges. This will help me rely less on tutorials and become more confident in analyzing files and identifying the right tools or techniques to use when solving CTF problems.
